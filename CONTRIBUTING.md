# 🤝 Guía de contribución

> ¡Gracias por tu interés en contribuir al **FC Mobile Info Bot**!  
> Esta guía te ayudará a empezar.

---

## 📋 Cómo contribuir

### 🐛 Reportar errores

Si encuentras un error:

1. ✅ Verifica que no exista ya un [issue abierto](../../issues)
2. 📝 Crea un nuevo issue con:
   - Descripción clara del problema
   - Pasos para reproducirlo
   - Comportamiento esperado vs actual
   - Versión de Python y sistema operativo

### 💡 Sugerir mejoras

1. 📂 Abre un issue con la etiqueta `enhancement`
2. 📝 Describe la mejora propuesta
3. 💭 Explica por qué sería útil

### 💻 Enviar código

**1️⃣** 🍴 Haz **fork** del repositorio

**2️⃣** 📥 **Clona** tu fork localmente

**3️⃣** 🌿 Crea una **rama** para tu cambio:

```bash
git checkout -b feature/descripcion-corta
```

**4️⃣** ✏️ Realiza tus **cambios**

**5️⃣** 🧪 **Prueba** que todo funcione correctamente

**6️⃣** 💾 **Commit** con mensajes descriptivos:

```bash
git commit -m "feat: agrega soporte para X"
```

**7️⃣** 📤 **Push** a tu fork:

```bash
git push origin feature/descripcion-corta
```

**8️⃣** 📝 Abre un **Pull Request** hacia la rama `main`

---

## 📐 Convenciones de código

| Convención | Descripción |
|------------|-------------|
| 🌐 **Idioma** | Usa español para comentarios y documentación |
| 📏 **Estilo** | Sigue el estilo PEP 8 para Python |
| 📦 **Funciones** | Mantén las funciones cortas y enfocadas |
| 📚 **Docstrings** | Añade docstrings a funciones nuevas |

---

## 📝 Estructura de commits

Prefijos sugeridos:

| Prefijo | Uso |
|---------|-----|
| `feat:` | ✨ Nueva funcionalidad |
| `fix:` | 🐛 Corrección de error |
| `docs:` | 📚 Cambios en documentación |
| `refactor:` | ♻️ Refactorización de código |
| `style:` | 💅 Cambios de formato (sin cambio de lógica) |

---

## ❓ Preguntas

Si tienes dudas, abre un issue con la etiqueta `question`. 👋
